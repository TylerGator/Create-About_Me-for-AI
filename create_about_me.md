## About This File

This file helps you build a persistent AI context profile — a snapshot of who you are, how you work, and how you'd like an AI assistant to help you. Once complete, you can load it at the start of any AI session so the assistant understands your role and preferences without you having to explain them each time.

**To get started:** Share this file with an AI assistant and it will interview you. Or, fill in the sections yourself and hand it back to the AI to refine.

# Profile Monitoring & Update Protocol
<!-- ============================================================
     FOR AI USE IN PROFILE MONITORING MODE ONLY
     Activate this when the profile is loaded (Scenario B).
     Do not display this section to the user unless asked.
     ============================================================

     WHAT THIS IS:
     While helping the user with their work, quietly monitor the
     conversation for signals that their profile may need updating.
     At a natural break point, surface a single, brief question
     asking if the profile should be amended. Never interrupt
     mid-task to do this — wait for a natural pause.

     ---

     SIGNALS THAT SUGGEST AN UPDATE IS NEEDED:
     Watch for any of the following during the session:

     Role / Responsibilities:
     - User mentions a new project, initiative, or engagement not
       referenced in section 1.2 or 1.3
     - User describes a responsibility that contradicts or extends
       what is listed in their profile
     - User indicates a change in title, scope, or seniority

     Key Relationships:
     - User introduces or references a person not listed in
       section 1.4 (new direct report, new peer, new stakeholder)
     - User describes a changed relationship (e.g., a former peer
       is now their supervisor; a key contact has left)

     Working Style / Tone:
     - User corrects an AI response with "actually I prefer..." or
       "that's not quite how I work..."
     - User's written instructions or edits during the session
       noticeably differ from the tone described in section 2.1
     - User expresses a consistent preference that isn't captured
       in section 2.2 or 2.3

     Priorities and Themes:
     - User spends significant session time on a topic not listed
       in their recurring themes or top responsibilities
     - A major effort, project, or engagement has wrapped up and
       new priorities have taken its place

     Organizational Context:
     - User references a significant change (reorg, leadership
       change, budget shift, new policy mandate) not in section 3.1

     Profile Gaps:
     - The AI had to ask a clarifying question mid-session that a
       complete profile should have already answered

     ---

     WHEN TO SURFACE THE UPDATE QUESTION:
     Trigger the update check at ONE of the following moments:
     - When the user signals they are wrapping up ("that's all,"
       "thanks," "I think we're done")
     - After a major deliverable is completed (memo drafted,
       briefing prepared, document finalized)
     - When 2 or more signals have been detected in the session
       and there is a natural pause in the conversation
     
     Never ask during active work. One check per session is enough
     unless the user initiates further updates themselves.

     ---

     HOW TO SURFACE THE UPDATE QUESTION:
     Keep it brief and non-intrusive. Use this format:

     "Before we finish — I noticed [brief, specific observation,
     e.g., 'you mentioned a new working group with the Office of
     Finance that isn't in your profile' / 'you adjusted the tone
     of the memo in a way that differs slightly from your current
     writing preferences']. Would you like me to suggest an
     amendment to your about_me.md to capture that?"

     If the user says yes, proceed to AMENDMENT GENERATION below.
     If the user says no or not now, acknowledge and close cleanly.
     Do not ask again in the same session.

     ---

     AMENDMENT GENERATION:
     When the user agrees to an update:

     1. Identify the specific section(s) to change (e.g., "1.4 —
        Key People and Relationships" or "2.1 — Your Writing Tone").

     2. Show the proposed change clearly, using this format:

        SECTION: [section number and title]
        CURRENT TEXT: [quote the existing text, or "blank / not yet filled"]
        PROPOSED UPDATE: [the new or amended text]
        REASON: [one sentence explaining why this change was suggested]

     3. Ask: "Does this look right, or would you like to adjust
        the wording before I finalize it?"

     4. Once confirmed, generate the full amended text block the
        user can copy into their about_me.md file directly.

     5. Provide a Change Log entry the user can paste into the
        Profile Change Log section, using this format:

        | [Today's date] | [Section name] | [One-line summary of change] |

     ---

     TONE GUIDANCE FOR UPDATE CONVERSATIONS:
     - Be brief — the user is wrapping up, not starting a new task
     - Frame updates as capturing something useful, not as a
       correction or failure of the original profile
     - Never suggest more than 2 amendments in a single session;
       prioritize the most impactful change if several signals exist
     - If the user seems rushed, offer to summarize the suggested
       changes so they can decide later: "I can give you a quick
       summary to review when you have a moment."
     ============================================================ -->
